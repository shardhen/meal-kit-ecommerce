# Meal Kit E-commerce

A full-stack meal kit e-commerce web application built with Node.js, Express.js, EJS, MongoDB, and JavaScript.  
The application allows customers to browse meal kits, create an account, log in, add products to a shopping cart, and place orders. It also includes a clerk/admin role for managing meal kit products.

## Technical Functionality

### Full-Stack Web Application
This project is built as a full-stack web application using Node.js, Express.js, EJS, MongoDB, and Bootstrap. The application follows a server-side rendering approach with EJS templates and Express routes to handle user interactions, meal kit browsing, authentication, cart management, and product administration.

### User Authentication and Session Management
The application includes user sign-up and log-in functionality. User information is stored in MongoDB, and sessions are managed using `express-session`. After logging in, users are assigned a role such as `customer` or `clerk`, allowing the application to display different features depending on the user type.

### Role-Based Access Control
The project uses role-based access control to separate customer and clerk functionality.

- Customers can browse meal kits, add items to the shopping cart, and place orders.
- Clerks can access product management features, including adding, editing, and deleting meal kits.

This helps demonstrate basic authorization logic in a real web application.

### MongoDB and Mongoose Integration
MongoDB Atlas is used as the cloud database for storing user and meal kit data. Mongoose is used to define schemas and interact with the database.

The database supports:

- Storing registered user information
- Loading and retrieving meal kit data
- Adding new meal kits
- Editing existing meal kit information
- Deleting meal kits
- Managing product details such as title, description, category, price, servings, cooking time, featured status, and image path

### Meal Kit Product Management
The application includes clerk/admin product management functionality. A clerk can create, update, and delete meal kit products through web forms. This makes the project more than a static shopping page and demonstrates CRUD operations using Express routes and MongoDB.

### Image Upload Functionality
The project uses `express-fileupload` to allow clerks to upload meal kit images. Uploaded images are saved to the server’s public image directory, and the image path is stored in MongoDB along with the meal kit data. This allows newly added or updated products to display their uploaded images dynamically on the website.

### Shopping Cart Functionality
The application includes a session-based shopping cart. Customers can add meal kits to the cart, view selected items, remove items, and place an order. Cart data is stored in the user session while the user is active.

The cart calculates:

- Item quantity
- Subtotal
- Tax
- Final total

### Mailgun Email Integration
Mailgun is integrated to send transactional emails. The application sends a welcome email after user registration and sends an order confirmation email after a customer places an order. This demonstrates the use of a third-party email API in a full-stack application.

### Error Handling
The application includes basic error handling for unauthorized access, missing pages, and server-side issues. For example, users who are not logged in or do not have the correct role are redirected or shown an error page.

### Deployment with Vercel
The project is deployed using Vercel, allowing the application to be accessed through a live web URL. Environment variables are used to manage sensitive configuration values such as the MongoDB connection string, session secret, and Mailgun API key.
